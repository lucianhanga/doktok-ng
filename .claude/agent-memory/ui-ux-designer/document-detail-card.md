`apps/ui/src/DocumentDetail.tsx` renders a document's detail card. Sections in order: result-head (Back button + h2 title), status `<dl>` (File / Type / Status / Pages), Processing table (features + Retry), Entities chips, Content `<pre>` (extracted content.md), Activity timeline. Receives `id` and `onClose`. Sibling components navigate between docs via an `onOpenDocument(id)` callback pattern (used for the duplicate "Open original" action).

Document model (`DokDocument` in api.ts): `id`, `original_filename`, `detected_mime`, `title`, `status` (active|failed|duplicate|quarantined), `created_at`, `metadata`. Backend Document also has `storage_path` and `duplicate_of` (original id for duplicates) — these may need surfacing in the API response/types for the file-actions feature.

File-actions feature (designed 2026-06): three actions on the card — (1) "Open original" shown only for duplicates, navigates via onOpenDocument(duplicate_of); (2) "Open in new tab" -> new endpoint serving raw bytes; (3) "Preview" -> in-app modal/overlay. Requires new backend endpoint `GET /api/v1/documents/{id}/file?variant=original|normalized` serving inline bytes with correct Content-Type, Content-Disposition, and X-Content-Type-Options: nosniff. Status indicator convention: green=success, red=error/failure, yellow=warning.

Detail-view redesign (designed 2026-06-12, design-only, not yet implemented): old layout = 7 equal-weight stacked sections, felt like an undifferentiated wall. Chosen pattern: **sticky identity header (Back + title + status PILL + Preview/Open/Download) + two-column main+aside body, with Content/Entities/Activity behind WAI-ARIA tabs**. Main = Summary then the tab strip; aside (~300px, stacks FIRST on <=860px) = metadata `<dl>` (Status REMOVED from it, now the header pill), Categories, compact Processing list with a colored roll-up line ("1 feature failed") + inline Retry on non-done features. Entities tab = grouped by entity_type, per-group cap + "Show all (n)", total count in the tab label. Content tab = the existing height-bounded `.content` <pre>. **Lazy-load rule (hand-off to API agent reviewing the 6-endpoint fan-out):** `/content`, `/entities`, `/audit` fetch only on first tab activation; `document`+`categories`+`features`+cheap entity/activity counts should be eager and ideally consolidated into one `GET /documents/{id}?include=features,categories,counts` so identity/aside paint without waiting on heavy payloads. Empty primary sections (Summary/Content) show muted copy; empty optional aside (Categories) may hide; Processing always renders. Skeleton (not spinner) on initial load; focus -> Back on open, return to originating list row on close; Escape = Back unless preview modal open. Reuses existing CSS: `.tabs`, `.content`, `.feature-chips`, `.badge.status-*`, `.banner-warning` — zero new deps.
